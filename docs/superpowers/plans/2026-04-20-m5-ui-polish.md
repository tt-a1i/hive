## Chunk 3: 新增依赖
为实现规范的设计系统，需要向 `web/package.json` 新增以下运行时/构建依赖（不影响 server 端）：
- **`tailwindcss` & `@tailwindcss/vite` (v4)**：提供快速的原子化 CSS。v4 采用全新的 CSS-first 配置，通过 `@import` 和 `@theme` 直接驱动。
- **`lucide-react`**：标准化的矢量图标库，提供一致的线框风格，符合 kanban 工具的视觉调性。
- **`clsx` & `tailwind-merge`**：优雅地合并动态 Tailwind class，解决组件条件渲染时的类名冲突（如 `cn()` utility）。
- **`@radix-ui/react-dialog` & `@radix-ui/react-tooltip`**：高质量的 Headless UI 组件，提供开箱即用的无障碍（a11y）支持、焦点陷阱和键盘导航。

## Chunk 4: 设计 token 系统
在 `web/src/styles/globals.css` 中注入以下全局变量（基于 Tailwind v4 `@theme` 扩展）：
- **字体**：默认无衬线字体族（Inter / SF Pro），代码和终端严格使用 Monospace（JetBrains Mono 或系统等宽）。
- **Spacing Scale**：沿用 Tailwind 默认的 `0.25rem` 步进（p-2, p-4, gap-3 等）。
- **颜色 (Surface/Text)**：
  - `surface-0` 到 `surface-4`（深色主导，`#1F2428` 等）用于侧边栏、主区、卡片的分层。
  - `--color-border` (`#30363D`) 用于面板分割线。
  - `text-primary` (高亮), `text-secondary` (描述), `text-tertiary` (禁用)。
- **颜色 (Status)**：`status-green` (working), `status-orange` (idle), `status-red` (stopped/error)。
- **双模式 (Light/Dark)**：定义全套明暗双色 Surface 和 Text token。支持跟随系统或手动热切，确保深色酷炫且浅色高对比度可用。
- **Z-Index 层级**：`z-0` (基础), `z-10` (Sticky Headers/Tabs), `z-50` (Dropdowns/Tooltips), `z-100` (Modals/Overlays)。

## Chunk 6: Layout 重做
重构基础布局结构，彻底告别现在的流式排版：
- **全局容器**：全屏 `h-screen w-full flex overflow-hidden bg-surface-0 text-text-primary`。
- **Sidebar (左栏)**：固定宽度 `w-64`，右侧边框 `border-r border-border`，含 Workspace 列表与新建按钮。
- **主区域**：`flex-1 flex flex-col min-w-0`。
- **Top Bar**：固定高度 `h-14`，底边框，包含当前 Workspace 名称、全局状态以及 Tasks/Settings 入口。
- **内容区 (Split View)**：
  - **中栏 (Orchestrator Terminal)**：宽度 `w-[40%]` 或 `w-96`，`border-r border-border`，终端 Tab 置于顶部。
  - **右栏 (Worker Grid)**：剩余空间 `flex-1`，内边距 `p-4`，允许内部独立垂直滚动。
- **响应式策略**：桌面端优先。宽度 `< 1024px` 时，将三栏转为堆叠（Stack）或隐藏 Sidebar 依靠汉堡菜单唤出；中右两栏改为上下排列。

## Chunk 8: Worker 卡片
将现有的文本列表 `Worker cards coming next` 替换为真实的卡片网格。
- **布局**：`bg-surface-1 border border-border rounded-lg p-4 flex flex-col gap-2 hover:bg-surface-2 transition-colors`。
- **字段展示**：
  - 顶部：头像/Icon + **Name** (`Alice`) + **Role** Badge (`[Coder]`)。
  - 状态行：呼吸圆点 + **Status** 文本 (`● working` / `○ idle` / `🔴 stopped`)。
  - 摘要行：显示最近一条交互的截断文本，辅助判断当前进度。
  - 数据：**Pending Task Count** 以醒目标签显示（如 `队列: 2`）。
  - 底层：**Last Session ID** 作为次级灰色文本 (`text-xs text-text-secondary truncate`) 附在底部。
- **操作 (Hover / Footer Actions)**：
  - 卡片底部提供图标按钮组（基于 lucide）：`[Restart]`、`[Stop]`、`[View Terminal]`。点击 View Terminal 弹出抽屉或全屏模态框展示该 worker 的 PTY。

## Chunk 10: 开放问题
以下问题需要在真正实施前由用户拍板决定：

- **Tailwind v4 vs v3 vs CSS Modules**
  - *Resolved*: 拍板使用 Tailwind v4。
- **是否引入 Radix UI**
  - *Resolved*: 拍板引入 Radix UI，要求“尽可能完善”以解决无障碍和复杂弹窗交互。
- **深色模式 (Dark Mode)**
  - *Resolved*: 拍板双模式（Light/Dark），不凑合单模，需提供完整的主题无缝热切体验。
- **设计参考风格**
  - *Resolved*: 拍板使用 **Linear 风格**。UI 视觉上需要体现 clean、minimalist，边框极细，阴影与发光效果克制且精致，比 agent-kanban 原本的默认形态更有质感。
- **响应式布局策略**
  - *Resolved*: 拍板 **Web / Desktop Only**，放弃 Mobile / 触屏/ 小屏堆叠的支持。重心全放在优化宽屏生产力工具的快捷键、多栏布局和高密度展示上。
- **国际化 (i18n)**
  - *Current inclination*: 继续硬编码中英混排字符串（如 `文件已在外部变化`），MVP 不做字典提取。
  - *Needs user decision*: 是否借着重构 UI 的机会引入 i18n 方案？
- **视觉测试策略**
  - *Current inclination*: 维持现状，只做 DOM 级别的 `toHaveClass` 或存在性断言，不引入 Playwright 截图对比，避免 CI 变慢和维护开销。
